Create the migration file with exact content:

1. **categories table:**
```sql
create table public.categories (
  id           uuid primary key default gen_random_uuid(),
  household_id uuid not null references public.households on delete cascade,
  name         text not null,
  position     integer not null default 0,
  created_at   timestamptz default now()
);
create index on public.categories(household_id);
create index on public.categories(household_id, position);
```

2. **stores table:**
```sql
create table public.stores (
  id           uuid primary key default gen_random_uuid(),
  household_id uuid not null references public.households on delete cascade,
  name         text not null,
  created_at   timestamptz default now()
);
create index on public.stores(household_id);
```

3. **store_layouts table:**
```sql
create table public.store_layouts (
  store_id    uuid not null references public.stores on delete cascade,
  category_id uuid not null references public.categories on delete cascade,
  position    integer not null default 0,
  primary key (store_id, category_id)
);
create index on public.store_layouts(store_id, position);
```

4. **Add category_id to list_items:**
```sql
alter table public.list_items
  add column category_id uuid references public.categories on delete set null;
create index on public.list_items(category_id);
```

5. **RLS policies for categories** (4 policies: select/insert/update/delete using `household_id = public.my_household_id()`):
```sql
alter table public.categories enable row level security;
create policy "categories_select" on public.categories for select using (household_id = public.my_household_id());
create policy "categories_insert" on public.categories for insert with check (household_id = public.my_household_id());
create policy "categories_update" on public.categories for update using (household_id = public.my_household_id());
create policy "categories_delete" on public.categories for delete using (household_id = public.my_household_id());
```

6. **RLS policies for stores** (same 4-policy pattern):
```sql
alter table public.stores enable row level security;
create policy "stores_select" on public.stores for select using (household_id = public.my_household_id());
create policy "stores_insert" on public.stores for insert with check (household_id = public.my_household_id());
create policy "stores_update" on public.stores for update using (household_id = public.my_household_id());
create policy "stores_delete" on public.stores for delete using (household_id = public.my_household_id());
```

7. **RLS policies for store_layouts** (scoped via stores join — store_id must belong to household):
```sql
alter table public.store_layouts enable row level security;
create policy "store_layouts_select" on public.store_layouts for select
  using (store_id in (select id from public.stores where household_id = public.my_household_id()));
create policy "store_layouts_insert" on public.store_layouts for insert
  with check (store_id in (select id from public.stores where household_id = public.my_household_id()));
create policy "store_layouts_update" on public.store_layouts for update
  using (store_id in (select id from public.stores where household_id = public.my_household_id()));
create policy "store_layouts_delete" on public.store_layouts for delete
  using (store_id in (select id from public.stores where household_id = public.my_household_id()));
```

8. **Realtime publication — categories ONLY** (stores and store_layouts do NOT need realtime; store selection is session-only):
```sql
alter publication supabase_realtime add table public.categories;
```
