# Plan 20-01: Phase 20 Backend and Query Infrastructure

Update database functions and TypeScript query definitions to support brand and image URL persistence and retrieval.

## 🛠️ Proposed Changes

### 💾 Database Migrations
- Create `supabase/migrations/20260316000000_phase20_sync_enrichment.sql`:
    - Update `public.upsert_household_item_memory` to accept `p_brand` and `p_product_image_url` and use `COALESCE` in the `ON CONFLICT` clause.
    - Update `public.sync_household_item_memory` to pass `new.brand` and `new.product_image_url` from `list_items` to `upsert_household_item_memory`.
    - Update `public.search_household_item_memory` to return `brand` and `product_image_url`.

### 🧬 TypeScript Queries
- `src/lib/queries/items.ts`:
    - Update `Item` type to include `brand?: string | null` and `product_image_url?: string | null`.
    - Update `AddItemVariables` to include `brand?: string | null` and `imageUrl?: string | null`.
    - Update `createAddItemMutation` to insert `brand` and `product_image_url` and select them back.
    - Update `createAddOrIncrementItemMutation` to insert these fields on the new-row path.
- `src/lib/queries/item-memory-admin.ts`:
    - Update `ItemMemoryEntry` type.
    - Update `UpdateItemMemoryVariables` type.
    - Update `createItemMemoryQuery` to select `brand` and `product_image_url`.
    - Update `createUpdateItemMemoryMutation` to handle `brand` and `product_image_url` updates.

## 🧪 Verification Plan

### ⚙️ Automated Tests
- Run `npm run test:rls` to ensure migration hasn't broken row-level security.
- Create a temporary script or use `execute_sql` to verify `upsert_household_item_memory` correctly stores brand and image.

### 🧤 Manual Verification
- Manually trigger an insert into `list_items` with a brand and image URL and verify the `household_item_memory` row is updated via `sync_household_item_memory`.
- Inspect the returned data in the shopping list query using DevTools to ensure `brand` and `product_image_url` are present.
