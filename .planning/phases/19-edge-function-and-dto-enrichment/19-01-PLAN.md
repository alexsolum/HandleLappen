# Plan 19-01: Edge Function and DTO Enrichment

Enrich the barcode scanning pipeline with brand names and image URLs from Kassal.app and Open Food Facts. This plan focuses on the Edge Function logic, type definitions, and cache backfill strategy.

## 1. Type Definitions & Shared Logic

Update shared types and normalization utilities to support brand and image data.

- [ ] Update `BarcodeLookupDto` in `supabase/functions/_shared/barcode.ts`:
  - Add `brand: string | null`
  - Add `imageUrl: string | null`
- [ ] Update `BarcodeCacheRow` in `supabase/functions/_shared/barcode.ts`:
  - Add `brand: string | null`
  - Add `image_url: string | null`
- [ ] Update `ReducedProviderPayload` in `supabase/functions/_shared/barcode.ts`:
  - Add `imageUrl: string | null`
  - Update `providers.kassal` to include `image: string | null`
  - Update `providers.openFoodFacts` to include `image: string | null`
- [ ] Update `BarcodeLookupDto` and `isBarcodeLookupDto` in `src/lib/barcode/lookup.ts` to match shared types.
- [ ] Implement `isJunkBrand(brand: string | null): string | null` in `_shared/barcode.ts`.
  - Filters: "None", "N/A", "Ukjent", "n/a", etc.
- [ ] Implement `getOFFImage(product: OpenFoodFactsProduct): string | null` in `_shared/barcode.ts`.
  - Priority: `image_front_no_small_url` -> `image_front_small_url` -> `image_small_url` -> `image_thumb_url`.

## 2. Edge Function Implementation (`supabase/functions/barcode-lookup/index.ts`)

Update the Edge Function to fetch and process enriched data.

- [ ] Update `extractKassalProduct` to handle the `data.products[0]` structure (Kassal API v1).
- [ ] Update `Dependencies.readCache` to implement the **Activation Date Safeguard**:
  - `ACTIVATION_DATE = '2026-03-14T00:00:00Z'`
  - If `(data.brand === null || data.image_url === null)` AND `data.provider_fetched_at < ACTIVATION_DATE`, return `null` to trigger re-fetch.
- [ ] Update `buildReducedProviderPayload` in `_shared/barcode.ts` to extract `brand` (using `isJunkBrand`) and `imageUrl` from both providers.
- [ ] Update `fallbackLookupFromProviderPayload` and `applyGeminiResult` to carry through `brand` and `imageUrl`.
- [ ] Update `createCacheRow` to include `brand` and `image_url`.
- [ ] Ensure `enrichWithGemini` prompt EXCLUDES brand and image URLs to save tokens.

## 3. Client-Side DTO Mapping (`src/lib/barcode/lookup.ts`)

- [ ] Update `BarcodeSheetModel` to include `brand: string | null` and `imageUrl: string | null`.
- [ ] Update `mapBarcodeLookupResult` to pass these fields from DTO to model.
