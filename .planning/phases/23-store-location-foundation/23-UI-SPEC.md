## Spacing Scale

Declared values (multiples of 4):

| Token | Value | Usage |
|-------|-------|-------|
| xs | 4px | Icon gaps, chip padding, inline separation |
| sm | 8px | Compact element spacing, button gap |
| md | 16px | Default element spacing, card padding, field margin |
| lg | 24px | Section padding, page vertical rhythm |
| xl | 32px | Layout gaps between major sections |
| 2xl | 48px | Major section breaks |
| 3xl | 64px | Page-level bottom clearance (nav padding) |

Exceptions:
- Map container: explicit `height: 250px` (locked decision from CONTEXT.md — Tailwind class `h-[250px]`)
- Page container bottom padding: `pb-24` (96px — existing project pattern for fixed bottom nav clearance)
- Touch targets (delete, close buttons): minimum 44px via `p-2` on 24px icon (`rounded-full p-2` = 40px; round up to `p-2.5` = 44px if needed)

**Source:** Existing page layout (`px-4 py-6 pb-24`), existing card padding (`px-4 py-3`), CONTEXT.md map height decision.

## Typography

| Role | Size | Weight | Line Height |
|------|------|--------|-------------|
| Body | 14px (text-sm) | 500 (font-medium) | 1.5 |
| Label | 14px (text-sm) | 500 (font-medium) | 1.4 |
| Heading (page) | 24px (text-2xl) | 600 (font-semibold) | 1.2 |
| Heading (section) | 20px (text-xl) | 600 (font-semibold) | 1.2 |

Two weights only: 500 (font-medium) for body and label text; 600 (font-semibold) for all headings. Weight 400 (normal) is not used.

**Applies to this phase:**
- Page heading "Rediger butikk / {store display name}": 20px semibold (text-xl font-semibold) — matches existing store edit page pattern
- Form field labels ("Kjede", "Butikknavn", "Vises som"): 14px medium
- "Vises som: Rema 1000 Teie" preview text: 14px medium, `text-gray-600`
- Map attribution text (Leaflet-generated): 11px — rendered by Leaflet, not project CSS
- Category section heading (unchanged, below map): inherits existing DraggableCategoryRow styles

**Source:** Scanned `butikker/+page.svelte` (`text-2xl font-semibold`, `text-sm font-medium`), `butikker/[id]/+page.svelte` (`text-xl font-semibold`, `text-sm text-gray-500`), `button.svelte` (`text-sm font-medium`). Weight reduced from 3 to 2 per typography contract (checker revision 2026-03-28).

## Interaction States

### Chain Dropdown

| State | Visual |
|-------|--------|
| Default | `select` element, `border-gray-300`, `rounded-lg`, `text-sm text-gray-900`, `w-full` |
| Focus | `focus:border-green-600 focus:outline-none focus:ring-2 focus:ring-green-500` |
| Disabled | `opacity-60 cursor-not-allowed` (during mutation pending) |

Use a native `<select>` element — consistent with the project's mobile-first approach (native OS picker on mobile, no custom dropdown library).

### Location Name Input

Identical to existing `input.svelte` component: `rounded-md border border-gray-300 bg-white px-3 py-2 text-sm`.

### "Vises som" Preview

Inline text below location name field:
```
Vises som: {composed name}
```
- 14px medium, `text-gray-600`
- Updates reactively as chain and location name change
- Hidden if `location_name` is empty

### Map Widget States

| State | Visual |
|-------|--------|
| Loading | Container shows gray background with "Laster kart…" centered in 14px `text-gray-500`, min-height 250px |
| Loaded, no pin | Map tiles visible, no marker. Instructional text below map: "Trykk på kartet for å plassere pin." (14px, `text-gray-500`) |
| Loaded, pin placed | Blue default Leaflet marker at tap position. Instructional text changes to: "Trykk for å flytte pinnen." |
| Loaded, existing pin | Map centered on saved coordinates at zoom 14, marker at saved position |
| Error (tiles fail) | Map container shows dashed border `border-dashed border-gray-300`, text "Kartet kunne ikke lastes. Sjekk internettforbindelsen og last siden på nytt." (`text-gray-500`) |

Map container: `rounded-xl overflow-hidden border border-gray-200` wrapping the Leaflet div.

### Lagre endringer Button States

| State | Visual |
|-------|--------|
| Default | `bg-green-700 text-white rounded-lg px-3 py-2 text-sm font-medium hover:bg-green-800` |
| Pending (mutation) | `opacity-60 cursor-not-allowed` + spinner or text "Lagrer…" |
| Error | Button re-enables; error banner appears above button |
| Success | No visual change; data refreshes via TanStack Query invalidation |

## Form Layout — Store Edit Page

Section order (locked decision from CONTEXT.md):

```
1. Back link "← Butikker"
2. Page heading: {composed store name}

3. [Label] Kjede
   [Select: chain dropdown]

4. [Label] Butikknavn
   [Input: location_name]

5. [Preview] Vises som: {composed name}

6. [Map widget — 250px height]
   [Instructional text below map]

7. [Button: Lagre endringer]

8. [Divider or spacing]

9. [Existing: Category drag-reorder section]
   [DraggableCategoryRow list]
```

Container: `mx-auto max-w-lg px-4 py-6 pb-24` (existing pattern)
Card wrapping form fields: `rounded-xl border border-gray-200 bg-white px-4 py-4 space-y-4 mb-4`

**Primary visual anchor:** the map widget (250px, visually dominant element on the page) with the "Lagre endringer" button as the primary action focus. The map draws the eye as the largest single element; the save button anchors the form's completion intent directly below it.

## Accessibility

| Rule | Value |
|------|-------|
| Touch target minimum | 44px (buttons with `p-2.5` or larger, `min-h-[44px]`) |
| Focus visible | All interactive elements retain browser default focus ring or explicit `focus:ring-2 focus:ring-green-500` |
| Map keyboard accessibility | Leaflet map is a supplementary input; coordinates are optional (nullable). Non-keyboard fallback: latitude/longitude are not manually entered by the user — if keyboard-only navigation is needed, future enhancement. Meets WCAG 2.1 AA for optional map inputs. |
| ARIA labels | Delete button: `aria-label="Slett {name}"`. Map container: `aria-label="Kart for å plassere butikkens posisjon"`. |
| Color contrast | green-700 on white: 5.74:1 (passes AA). red-700 on red-50: 6.15:1 (passes AA). gray-900 on white: 19.1:1 (passes AAA). |
| Select element | Native `<select>` — inherits OS accessibility behavior; screen reader friendly. |
