# Phase 14-02: Recipe List & Creation Flow

## Goal
Implement the recipe list view (`/oppskrifter`) and the creation flow (`/oppskrifter/ny`) with image upload and ingredient suggestions.

## Plan
1.  **Recipe Store/Queries**:
    *   Create `src/lib/queries/recipes.ts`:
        *   `createRecipesQuery`: Fetch all recipes for household.
        *   `createRecipeMutation`: Insert recipe + upload image (if present) + insert ingredients.
        *   `createDeleteRecipeMutation`: Delete recipe + associated image.

2.  **Ingredient Builder Component**:
    *   Create `src/lib/components/recipes/IngredientBuilder.svelte` (or reuse/adapt `ItemInput.svelte`).
    *   Features: Typeahead suggestions from `household_item_memory`, normalized naming, add button, list of added ingredients with delete option.

3.  **Image Upload Utility**:
    *   Create `src/lib/storage/upload.ts`:
        *   Client-side compression (canvas -> WebP, max 1200px).
        *   Supabase storage upload logic (bucket `recipes`, path `{household_id}/{uuid}.webp`).

4.  **Creation Page (`/oppskrifter/ny`)**:
    *   Form: Name (text), Description (textarea), Image (file input with preview), Ingredients (builder).
    *   Submit logic: Upload image -> Create recipe -> Add ingredients -> Redirect to `/oppskrifter`.

5.  **List Page (`/oppskrifter`)**:
    *   Display recipes in a grid (large cards) or list (depending on density decision).
    *   Show image (or placeholder), name, and ingredient count.
    *   Implement search bar filtering.
    *   "Add Recipe" FAB/button.

## Testing Strategy
*   **Playwright**:
    *   Test creation flow: Add name, upload dummy image, add ingredients, submit.
    *   Test list view: Verify created recipe appears with correct details.
    *   Test search: Filter list by name.
