# Phase 14-04: Edit Recipe

## Goal
Implement the edit recipe view (`/oppskrifter/[id]/rediger`), allowing updates to name, image, and ingredients.

## Plan
1.  **Edit View (`/oppskrifter/[id]/rediger`)**:
    *   **Pre-fill**: Fetch recipe details and populate form fields (Name, Description, current Image, Ingredients).
    *   **Image Handling**:
        *   Show current image with "Change" button.
        *   If new file selected -> compress -> upload (overwrite or new path?).
        *   If removed -> clear `image_url`.
    *   **Ingredient Handling**:
        *   Display current ingredients in builder.
        *   Allow reordering (drag/drop or up/down - maybe strict ordering not critical for MVP, just list management).
        *   Allow adding new/deleting existing.

2.  **Update Mutation**:
    *   Create `updateRecipeMutation`:
        *   Update `recipes` fields.
        *   Sync `recipe_ingredients`:
            *   Compare current list vs submitted list.
            *   Insert new.
            *   Delete missing.
            *   Update existing (if name/position changed).

3.  **Validation**:
    *   Ensure non-empty name.
    *   Ensure at least one ingredient (maybe optional?).

## Testing Strategy
*   **Playwright**:
    *   Test update flow: Change name, replace image, add/remove ingredients.
    *   Verify changes persist on detail view.
    *   Verify image updates in storage (via URL change or similar).
