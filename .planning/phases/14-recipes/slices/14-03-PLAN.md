# Phase 14-03: Recipe Detail & Add to List

## Goal
Implement the recipe detail view (`/oppskrifter/[id]`) and the ability to add selected ingredients to a shopping list.

## Plan
1.  **Detail Page Query**:
    *   Create `createRecipeQuery(id)`: Fetch recipe + ingredients (ordered by position).
    *   Create `createListsQuery` (reuse): Fetch household lists for the picker.

2.  **Detail View Component**:
    *   Header: Image (full width/large), Name, Edit/Delete actions (overflow menu).
    *   Ingredients List: Checkboxes for selection (default checked?), nice typography.
    *   "Add to List" FAB/Bottom Action Bar.

3.  **"Add to List" Logic**:
    *   **List Picker**: Bottom sheet/modal to select target list (default to active/last used if possible, or force choice).
    *   **Mutation**: Reuse `createAddOrIncrementItemMutation` (or batch version).
        *   Iterate selected ingredients.
        *   Call `addOrIncrementItem` for each (handling quantity update).
    *   **Feedback**: Show toast "Added [N] items to [List Name]". Stay on recipe page.

4.  **Edit/Delete Actions**:
    *   Link to `/oppskrifter/[id]/rediger`.
    *   Delete confirmation dialog -> delete mutation -> redirect to list.

## Testing Strategy
*   **Playwright**:
    *   Test detail load: Verify name, image, ingredients visible.
    *   Test selection: Toggle ingredient checkboxes.
    *   Test add to list: Select list, verify items appear in that list (incrementing if duplicate).
    *   Test delete: Verify recipe removed from list and DB.
