  <files>supabase/functions/barcode-lookup/index.ts, supabase/functions/barcode-lookup/index.test.ts, supabase/functions/barcode-lookup/deno.json, supabase/functions/_shared/barcode.ts</files>
  <action>
Implement an authenticated Supabase Edge Function named `barcode-lookup` that:
1. Validates JWT auth and request body `{ ean, listId? }`.
2. Normalizes the barcode to digits only and rejects impossible lengths early.
3. Checks `barcode_product_cache` for a fresh row and returns it immediately when valid.
4. Calls Kassal `/api/v1/products/ean/{ean}` with Bearer token secret.
5. Calls Open Food Facts only when Kassal is empty or insufficient.
6. Builds one reduced provider payload and sends that to Gemini using structured JSON output.
7. Validates Gemini output against the canonical category enum.
8. Stores found and not-found results with TTLs (found: 30 days, not-found: shorter).
9. Returns a small DTO the client can safely render.

Use shared helpers in `_shared/barcode.ts` for:
- canonical category enum
- barcode normalization
- provider merge rules
- response shaping
- Gemini schema validation

Add `supabase/functions/barcode-lookup/index.test.ts` with executable Deno coverage for:
- cache hit returning normalized DTO
- Kassal hit path
- Kassal miss plus Open Food Facts fallback hit
- provider miss producing one normalized not-found response
- Gemini structured-output schema validation and canonical-category rejection/fallback behavior

Do not expose provider payloads, tokens, or provider-specific failures in the response body.
