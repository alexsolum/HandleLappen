Locked UX decisions:
- Lookup should start automatically after scan detection.
- Scan and manual EAN are entry methods into one shared lookup-and-confirm tool.
- Result should feel like current bottom-sheet flows, not a detached mini-app.
- The user should never see two conflicting provider states.

Lookup contract from Plan 04-01:
```json
{
  "ean": "7044610878304",
  "found": true,
  "itemName": "Pepsi Max 1,5 L",
  "canonicalCategory": "drikke",
  "confidence": 0.92,
  "source": "kassal+gemini"
}
```

Category mapping rule:
- Resolve the returned canonical category key against the household's existing Phase 3 categories.
- If no match exists, allow add with no category and let the item fall into `Andre varer`.
