# Phase 13-02: Husstand Subpage Port

## Goal
Port content from `/husstand` to `/admin/husstand`, add breadcrumb, remove logout button, and implement server-side data loading.

## Plan
1. Create `src/routes/(protected)/admin/husstand/+page.server.ts`:
   - Fetch `members` (all profiles in the current household).
   - Fetch `invite_code` (from `households` for current `householdId`).
   - Use `locals.supabase` for fetching.
   - Return `members`, `inviteCode`, and `currentUserId`.
2. Update `src/routes/(protected)/admin/husstand/+page.svelte`:
   - Use content from `src/routes/(protected)/husstand/+page.svelte`.
   - Add `← Admin` breadcrumb above the `h1`.
   - Remove the `logout` section from the footer.
   - Adjust styling as needed to match the brand.
3. Remove `src/routes/(protected)/husstand/+page.svelte` (as it's now redundant).

## Testing Strategy
- Manual verification: Navigate to `/admin/husstand`, ensure it displays members and invite code, and the "← Admin" link works.
- Playwright test: In `tests/admin.spec.ts`, check that `/admin/husstand` works and contains correct info.
