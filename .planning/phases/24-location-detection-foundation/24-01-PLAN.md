Create `src/lib/location/session.svelte.ts` as a rune-store module following the `offline.svelte.ts` singleton pattern. Export a single session object plus imperative functions; do not request location on import.

Required exports and exact constants:

- `export const LOCATION_NORMAL_POLL_MS = 60_000`
- `export const LOCATION_RETRY_POLL_MS = 12_000`
- `export type LocationSessionStatus = 'idle' | 'explaining' | 'locating' | 'active' | 'permission-denied' | 'unavailable'`
- `export const locationSession = $state({ status: 'idle', detectedStoreId: null, lastFailure: null, showSettingsHint: false, lastSample: null })`
- `export function beginLocationExplanation(): void`
- `export async function confirmAutomaticStore(stores: DetectableStore[]): Promise<void>`
- `export async function retryLocationDetection(stores: DetectableStore[]): Promise<void>`
- `export function cancelLocationExplanation(): void`
- `export function refreshLocationStores(stores: DetectableStore[]): void`
- `export function stopLocationSession(): void`

Behavior requirements:
- `confirmAutomaticStore(stores)` must call `getCurrentLocation()` immediately in the same click-triggered code path. Do not insert any other awaited work before that geolocation request.
- Polling must use chained `setTimeout`, not `setInterval`.
- After a successful sample, store the sample, resolve the nearest store via `findNearestDetectedStore`, set `locationSession.detectedStoreId`, move status to `'active'`, and schedule the next poll after `60_000` ms.
- For `position-unavailable` and `timeout`, set status to `'unavailable'`, keep the manual picker path untouched, schedule one retry after `12_000` ms, then return to the normal cadence.
- For `permission-denied`, set status to `'permission-denied'`. First denied result keeps `showSettingsHint = false`. If `retryLocationDetection` receives a second denied result in the same session, set `showSettingsHint = true`.
- Register exactly one `visibilitychange` listener while polling is active. Hidden state clears the pending timer. Visible state triggers an immediate poll (`0` delay) before scheduling the next 60-second cycle.
- Guard against concurrent requests with a single `inFlight` boolean.
- `stopLocationSession()` must clear timers, remove the `visibilitychange` listener, reset `inFlight`, and leave `detectedStoreId` intact so the current selection survives route teardown.
  </action>
